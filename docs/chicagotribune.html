Chapter header

Add .chapter-header to any heading and wrap the content in a span.

chapter headline 

<h2 class="chapter-header">
  <span>Example chapter heading</span>
</h2>

Add a publication date

Add a publication date to the chapter header by wrapping the date in a small tag after the span.

chapter headline Published on a date with a time 

<h2 class="chapter-header">
  <span>Example chapter heading</span>
  <small>Example publication date</small>
</h2>

Related media

Links

Add the .media-link class to a a tag to apply the link styles. If the content is part of Digital Plus, use the digital-plus-icon class to add the plus icon to the end of the link.

This link goes elsewhere

<a href="..." class="media-link digital-plus-icon">...</a>

Linking to sidebar or other related content

Use aside for content that is related to the main article, such as a sidebar, and will display notched in the article. Use .related-content to style an overview of the sidebar. The .sublabel class can be used for additional description of the sidebar copy block.

Ads

How it works

Our current ad JS is located at http://media.apps.chicagotribune.com/ads/v1/ads.js. You're going to load that script and some dependencies, create one or more elements to contain the actual ads with the advert-wrapper class, make sure to serve an ad-iframe.html file from the same domain you're on, include a brief template somewhere on the page, run a line of JS and set the ad-path in order to get the right inventory displayed. Easy enough! (It's easier than it sounds, promise :)

Dependencies

You'll need to include the following, in order:

<script type="text/javascript" src="//cdnjs.cloudflare.com/ajax/libs/underscore.js/1.5.1/underscore-min.js"></script>
<script type="text/javascript" src="//cdnjs.cloudflare.com/ajax/libs/backbone.js/1.0.0/backbone-min.js"></script>
<script type="text/javascript" src="//media.apps.chicagotribune.com/ads/v1/ads.js"></script>

You'll also need to include an ad-iframe.html file served from the same domain your app will be running on. The easiest way to do this is probably to make a copy of that file and include it directly in your repo.

Create an ad container

This is just a straightforward div (or other DOM element) with the advert-wrapper class, meant to be the placeholder for the actual ad, which will get imported via Backbone and the ad-iframe.html file. For instance:

<div class="advert-wrapper"></div>

And if you'd like a cube ad instead:

<div class="advert-wrapper" data-ad-type="cube"></div>

Include a template

You need to include a brief template on the page that governs how the ads are presented (for instance, if you want to include the word "Advertisement" below each ad or something). The basic template looks like this: 

<script type="text/template" id="advert-template">
    <div class="advert-inner-wrapper">
        <iframe width="<%= width %>" height="<%= height %>" scrolling="no" frameBorder="0"></iframe>
    </div>
</script>

Load the ads and set the ad path

We're almost done! All you need to do now is wait for document.ready and run the following JS to load the ads and set the proper ad path (replacing with the correct one for your app, of course): 

$('.advert-wrapper').each(function() {
    new AdvertView({ el: $(this), path: '' });
});
