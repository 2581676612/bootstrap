Story body copy

Paragraphs within a .trib-article have a font-size is 16px, with a line-height of 26px and receive a margin-bottom of 22px. This is applied to all <p> tags within .trib-article elements. Body copy has a width of 680px at it's widest. width should be set using a percentage of the container holding the paragraphs to reach 680px.

Chapter/project header navigation

Fixed traveling header

The default "traveling" header is held in an element with ID #social-media. Once the user scrolls past this element, it sticks to top of the page, providing the user with social media buttons, page navigation, and project title.

The behavior is controlled by the {% setup_scripts %} block in the base Tribune Graphics template. This block also sets up "scrollspy" behavior, as seen in the next section.

{% block setup_scripts %}
<script type="text/javascript">
$.waypoints.settings.scrollThrottle = 30;
$('body').scrollspy({ offset: $('#social-media').height(), target: "#social-media"});
$('.advert-wrapper').each(function() {
 new AdvertView({ el: $(this), path: '{{ ad_path }}' });
});
$('#social-media').height($('#social-media').outerHeight());
new TravelerView({ traveler: $('#social-media'), target: $('#page-body') });
</script>
{% endblock %}

Line 9 is of particular interest, as it sets up the traveling behavior using TravelerView in the Tribune Graphics app.js. The traveler configuration option represents the element that will be fixed, the target option means that the traveler will be fixed when the target element is within the browser viewport.

Nav pills in the header

Wrap a regular ul of links with .nav and nav-pills and placed to the right of a .trib-title. Wrap everything with #social-media.

If the pills link to anchors on the page, the pills will automatically get the active class as the anchors scroll through the viewport. This is configured using Twitter Bootstrap's default "scrollspy" mechanism, and can be seen in line 4 above.
