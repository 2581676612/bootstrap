Story asset integration

Notched media

To integrate story assets with the main narrative, wrap a figure in a div with class .trib-media, which will notch the content into the story paragraphs. Wrap story body copy p elements and .trib-media content in an article with a ct-article class. This technique works best when all the content is within a none nested span12. Otherwise the width of the body copy p elements needs to be adjusted to line up with trib-media content.

Ads

How it works

Include the javascript, add <div class="advert"></div> where you want an ad to show up, specifiy a ad path and page type for the page, and presto!

<body data-ad-path="sports" data-ad-ptype="sf" data-ad-iframe-url="/static/ad-iframe.html">

<div class="advert"></div>

<div class="advert" data-ad-type="cube"></div>
<div class="advert" data-ad-type="cube"></div>

<script src="//ajax.googleapis.com/ajax/libs/jquery/1.10.1/jquery.min.js"></script>
<script src="//s3.amazonaws.com/media.apps.chicagotribune.com/ads/v2.0.0/ads.min.js"></script>
</body>

Best practice

Ads should be handled all together at once on a page. Ads shouldn't be treated as individual elements, but rather as an ad map or group of ads. Best practice is to include one leaderboard and two cube ads. The leaderboard should be at the top of the page, and the first cube placed on the the right side of the page, very close to the leaderboard. The second cube can be placed somewhere further down the page, on the right.

Unless you have different requirements that you've discussed with Ad Ops, you should follow this pattern.

Setup

You'll need to include this script, after your include for jquery:

<script src="//s3.amazonaws.com/media.apps.chicagotribune.com/ads/v2.0.0/ads.min.js"></script>

You'll need to include an ad-iframe.html file served from the same domain your app will be running on. The easiest way to do this is to make a copy of that file and include it the static assets directory of your app.

Configure the page

To get ads to serve properly, they need to know the ad path, page type, and where to find the ad iframe. Since these settings cannot vary ad to ad, we specify them on the body tag:

<body data-ad-path="sports" data-ad-ptype="sf" data-ad-iframe-url="/static/ad-iframe.html">

Ad paths

Ad paths sometimes resemble the URL path, i.e. sports/bears. You can always steal the path from an existing section on chicagotribune.com by inspecting an ad and looking at the url of the javascript tag that loads the ad. The ad path is the final bit of the path after the domain, account id and publication name. When in doubt, contact ad ops.

The ad path is always specifed in the body tag with the data-ad-path attribute.

Page type

There are two page type values: s for a story page and sf for a section front page. A story page is any page that displays a single content item, and a section front page is anything that displays a list or lists of stories or links. Always specify the page type in the body tag with the data-ad-ptype attribute.

Create your ad containers

This is just a straightforward div (or other DOM element) with the advert class, meant to be the placeholder for the actual ad:

<div class="advert"></div>

And if you'd like a cube ad instead:

<div class="advert" data-ad-type="cube"></div>

Refreshing ads

Sometimes you'll need to refresh the ads, if your dynamically changing the page contents without loading a new url

$('.advert').ad('refresh');

Refreshing ads will trigger special events that you can attach a listener to.

$('body').on('ad-refresh', function() {
  alert('Somewhere ads have been refreshed...');
})

Or for a specific ad container.

$('#my-special-ad').on('refresh', function() {
  alert('My special ad has been refreshed...');
})

No magic

The ads javascript will automatically load and initialize all elements with class .advert. If you want to use a different class name, or handle the loading yourself, you can use the $.ad() jQuery plugin.

$('.my-ads').ad({
  path: 'sports',                      // Optional: defaults to data-ad-path on the body tag
  ptype: 'sf',                         // Optional: defaults to data-ad-ptype on the body tag
  iframeUrl: '/static/ad-iframe.html', // Optional: defaults to data-ad-iframe-url on the body tag, then 'ad-iframe.html'
  type: 'cube',                        // Optional: defaults to data-ad-type on each selected element, then 'leaderboard'
  debug: true                          // Optional: defaults to false
});
